# 用例通过率与执行进度展示方案

## 一、数据来源

基于现有 `case_status` 字段计算：
- `0`: 未开始
- `1`: 通过
- `2`: 失败

## 二、统计维度

### 需求维度（Requirement 详情页）
- 显示该需求下所有用例的通过率

### 用例列表维度（CaseInfo 页面）
- 显示当前筛选/分组下用例的统计

## 三、展示位置

### 方案一：在 CaseInfo 页面工具栏区域显示

**位置**：搜索栏下方，工具栏左侧

```
┌────────────────────────────────────────────────────────────────┐
│  全部: 100  │  ✓ 通过: 80 (80%)  │  ✗ 失败: 15 (15%)  │  ◐ 未测: 5 (5%)  │
└────────────────────────────────────────────────────────────────┘
```

### 方案二：在搜索栏内作为统计信息显示

```
┌────────────────────────────────────────────────────────────────┐
│  搜索表单...                                                     │
├────────────────────────────────────────────────────────────────┤
│  总计 100 用例  │  ████████████████░░░░░░░░  80%  │  [查看详情]  │
└────────────────────────────────────────────────────────────────┘
```

### 方案三：在 ChoiceSettingArea 区域显示（推荐）

当前已有 `ChoiceSettingArea` 组件，可以在其基础上增加统计展示。

## 四、UI 设计

### 统计信息条

```
┌────────────────────────────────────────────────────────────────────────────┐
│                                                                            │
│  ████████████████████████████████████████████████░░░░░░░░░░░░  80/100     │
│                                                                            │
│  ✓ 通过: 60 (60%)         ✗ 失败: 15 (15%)         ◐ 未测: 25 (25%)     │
│                                                                            │
│  执行进度: 75% (75/100 已执行)                                              │
│                                                                            │
└────────────────────────────────────────────────────────────────────────────┘
```

### 元素说明

| 元素 | 描述 |
|------|------|
| 进度条 | 整体展示执行进度（已执行/总数） |
| 通过数 | case_status=1 的数量和占比 |
| 失败数 | case_status=2 的数量和占比 |
| 未测数 | case_status=0 的数量和占比 |
| 执行进度 | (通过+失败) / 总数 |

## 五、组件实现

### 新增组件：CaseStatsBar

```typescript
interface Props {
  total: number;           // 总用例数
  passed: number;          // 通过数 (status=1)
  failed: number;          // 失败数 (status=2)
  unchecked: number;       // 未测数 (status=0)
  showProgress?: boolean;  // 是否显示进度条
  onViewDetails?: () => void; // 查看详情回调
}
```

### 样式设计

- 使用主题色渐变进度条
- 绿色表示通过、红色表示失败、灰色表示未测
- 紧凑设计，适应工具栏高度

## 六、集成到 CaseInfo

在 `CaseInfo/index.tsx` 的 `CaseStepSearchForm` 和用例列表之间添加统计条：

```tsx
<CaseStepSearchForm ... />

<CaseStatsBar
  total={testCases.length}
  passed={passedCount}
  failed={failedCount}
  unchecked={uncheckedCount}
/>

<ProCard ...> /* 用例列表 */
```

## 七、计算逻辑

```typescript
const stats = useMemo(() => {
  const total = testCases.length;
  const passed = testCases.filter(tc => tc.case_status === 1).length;
  const failed = testCases.filter(tc => tc.case_status === 2).length;
  const unchecked = testCases.filter(tc => tc.case_status === 0).length;
  const executed = passed + failed;
  const passRate = executed > 0 ? Math.round((passed / executed) * 100) : 0;
  const executionRate = total > 0 ? Math.round((executed / total) * 100) : 0;

  return { total, passed, failed, unchecked, executed, passRate, executionRate };
}, [testCases]);
```
